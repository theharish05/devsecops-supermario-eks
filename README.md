##  Tech Stack & Rationale

*   **AWS (VPC, ECR, EKS):** Provides scalable network architecture, a secure private container registry, and a managed Kubernetes control plane.
*   **Terraform:** Infrastructure as Code (IaC) that ensures the underlying infrastructure is modular, trackable, and easy to destroy or scale.
*   **GitHub Actions:** Orchestrates the CI/CD lifecycle using encrypted secrets for secure, keyless-style authentication constraints.
*   **Trivy:** Acts as the automated DevSecOps security gate by scanning container image layers for CVEs (Common Vulnerabilities and Exposures) prior to deployment.
*   **Kubernetes (`kubectl`):** Manages pod lifecycles, configuration rollouts, and automatic provisioning of an external AWS LoadBalancer.

##  Getting Started

### 1. Infrastructure Provisioning
Navigate to the Terraform directory to initialize and deploy the cluster infrastructure:
```bash
cd EKS-TF
terraform init
terraform apply --auto-approve

```

### 2. Configure GitHub Secrets

Add the following credentials safely to your GitHub Repository Secrets (`Settings > Secrets and variables > Actions`):

* `AWS_ACCESS_KEY_ID`
* `AWS_SECRET_ACCESS_KEY`

### 3. CI/CD Deployment Flow

Pushing any changes to the `main` branch automatically triggers the `.github/workflows/deploy.yml` pipeline:

1. **Authenticates** with your AWS deployment user.
2. **Pulls and tags** the game application container image.
3. **Runs Trivy** to parse the image for `CRITICAL` or `HIGH` security vulnerabilities.
4. **Pushes** the validated image to Amazon ECR.
5. **Updates** your EKS cluster access entries and rolls out the latest deployment.

---

##  Accessing the Game

Once the GitHub Actions workflow successfully completes, fetch the public endpoint generated by the AWS Load Balancer:

```bash
kubectl get svc mario-v2-service

```

Copy the long DNS address found under the **`EXTERNAL-IP`** column and paste it into your browser to start playing!
